Claude Code Development Workflow

Plan → Isolate → Implement → Verify → Review (PR)

Step 1: Plan

Implementation plan
  • Analyze & outline the solution
  • Identify the impacted files
  • Create the docs/plans/…-design.mddocument

Step 2: Isolate

Git worktree
  • Create a feature branch
  • Use using-git-worktrees
  • Work dir: .worktrees/[feature]

Step 3: Implement

Code to plan
  • Implement changes, manage context
  • Build & style check
  • Run relevant tests

Step 4: Verify

Quality gates
  • Run tests & checks
  • Confirm behavior
  • Prepare PR notes

Step 5: Review (PR)

gh + code review
  • Create PR with gh
  • Checklist & test plan
  • Ask Claude to review based on the plan

Artifacts

Plan → docs/plans/<date>-<feature>-design.md
Branch/Dir → .worktrees/<feature>

Automations

PR via gh • Claude code review
Checks → build, tests, type-check

Outcomes

Isolation • Documentation • Verification • Traceability
Tip: For small, single-file fixes: skip worktrees, open a PR & assign Claude to review it
